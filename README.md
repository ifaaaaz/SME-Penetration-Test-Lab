# SME Penetration Testing Lab

## Overview

This project demonstrates a simulated penetration test conducted against a small office environment. The assessment followed a structured methodology including reconnaissance, vulnerability identification, exploitation planning, risk assessment, and remediation recommendations.

## Target Environment

* Simulated Windows Server 2016
* Simulated Ubuntu Linux Server
* Small Office Network Environment

## Tools Used

* Nmap
* Nikto
* Metasploit Framework
* Kali Linux

## Assessment Methodology

### Reconnaissance

Performed service discovery and version enumeration using Nmap to identify exposed services and potential attack surfaces.

### Vulnerability Assessment

Analyzed discovered services and identified outdated software versions that may be affected by publicly known vulnerabilities.

### Exploitation Planning

Configured Metasploit to demonstrate exploitation workflows for known vulnerabilities, including EternalBlue (CVE-2017-0144).

### Risk Analysis

Assessed the potential impact of identified vulnerabilities and documented business risks associated with successful exploitation.

## Key Findings

* Identified outdated OpenSSH service
* Identified outdated Apache service
* Demonstrated EternalBlue exploitation workflow
* Documented security weaknesses and attack paths
* Produced risk-rated remediation recommendations
